#1 Fonts

#1.1 Typography

Open Sans is the brand font used in all design applications. Its clean, simple design and variety of weights make it ideal for titles, headlines and paragraphs.

We will be using Open Sans for all text elements. Do not move away from this font family and apply additional fonts to specific elements.

#1.2 Headers

All HTML headings, <h1> through <h6>, are available.

Use .thin, .bold and .italic classes for thin, bold and italic text respectively.

Similarly use the .bold-italic class for both bold as well as italic text.

Use .upper class for upper-case text.

#3 Form Fields

#3.1 Fields

Individual form controls automatically receive global styling from bootstrap. All textual <input>, <textarea>, and <select> elements with .form-control are set to width: 100%; by default. Wrap labels and controls in form-group class for optimum spacing.

Be sure to include a <form> tag before adding <input> elements on a page.

For form fields, place a <label> tag followed by an <input> tag within a <div> tag. Ensure to use the form-group class for the div.

For <textarea> tags, be sure to mention the row attribute to specify the number of rows.

By Default, all form elements will have a width of 100%.

Example

For form elements to remain inline, provide the form with the following class form-inline. This will allow them to remain on the same line and will not move to the next line.

Example

To disable a form element, add the disabled attribute to the specific tag. A value can be specified but the presence of the attribute is enough to disable the field.

Example

#3.12 Help Text

Help text should be explicitly associated with the form control it relates to using the aria-describedbyattribute. This will ensure that assistive technologies – such as screen readers – will announce this help text when the user focuses or enters the control.

#4 Buttons

Use the button classes on an <a>, <button>, or <input> element.

If the <a> elements are used to act as buttons – triggering in-page functionality, rather than navigating to another document or section within the current page – they should also be given an appropriate role="button" attribute.

Always use the .btn base class and apply the additional classes (.btn-primary and .btn-secondary) along with it.

Example

Add the btn-lg class for large buttons, btn-sm for small buttons and btn-xs for extra-small buttons.

#7 Layout

Bootstrap uses columns and rows for its grid system. Think of a table, where columns represent vertical divisions and rows horizontal divisions.

When creating a layout, wrap your columns within a row class. Use the col-xs- class along with the number of columns (Maximum of 12) to add columns to that row.

For example, to create a 2 column layout, we will use two columns of col-xs-6 each (6 X 2 = 12).
